苦乐相随，值此年岁——2019年末随想

发表于 2020-01-19

缘由

第一次如此有仪式感地写年终总结，究其原因，一是今年不知为何朋友圈中圣诞、元旦节日气氛浓厚，见了不少人这一年来的经历和感想，触动不少，反思尤甚。聊天的时候和朋友说到感觉别人这一年的生活都很精彩，而自己似乎都显得和朋友圈格格不入，于是仔细回想了这一年以来发生的种种，有了这篇总结的素材。

此外，2019对我来说确是极其重要的一年，整个人不管是心理状态，追求的事物，眼中所见都在急剧地发生着变化，促使我不断地向前，不适应是常态。“不变的，只有变化”。趁着记忆还是温热，没有变成“永远的怀念”，记下几笔。

最能体现一个程序员修养的可能是他的github活动轨迹？

年初

这段时间是比较难受的一段时间，这时候我的情况是，虽然成绩一直在往上升，但是在加分和竞赛保研政策不明朗的情况下，保研还是八字没有一撇的事。

以平安夜为契机，我一边要拼命考好把成绩往上刷，一边还要想着寒假的时候做些什么来为增加自己今年9月份的时候工作或是推免的筹码，写了人生中第一份简历（虽然现在看来真的丑得一批而且没啥内容），投了以华为和字节为代表的几家公司的寒假实习，可都如泥牛入海，杳无音信。

期末考试结束后，我开始每天往蓝杰跑，一是想知道别人这个时候在干嘛，跟着大部队走也不至于掉队太多，二是多和别人接触也能得到更多的机会。正是在这时偶然间和鑫升、化楷他们一起组了队做区块链项目，抱着最后能落地盈利的想法，大家定下了每天的“上班时间”——上午十点到晚上十点。刚开始的时候，大家会的东西都很有限，就这么一点点地学习，一直坚持到五月份左右最终将我们的1.0版本“福币彩”给做了出来。在这个过程中每个人都牺牲了很多，也收获了很多，是我们第一次明白团队的意义所在。

下面这个截图就是我们的1.0版本啦

上半年学期

这个学期最重要的事情是为自己找一个尽量好的去向，不管是保研、考研还是工作，这个学期可以说是我翘课翘得最多的一个学期了，或者说就没怎么上过课，一来是因为知道自己听课不够专注等原因学到的东西很有限，二来是因为当时我们的项目还需要继续投入时间进行后续的工作。

大约在四月份的时候焦虑的心情达到了极点，那时候把能找到的实习生招聘入口都投了个遍，很多都是投了之后一点消息也没有，有消息的几家，商汤是投了简历大概半小时以后，HR打电话说马上把我的简历交给面试官安排面试，然后再也没有接到电话。阿里是投的当天晚上面试官和我约时间，当时想的是你们都这么高效率的嘛，我最近还没开始准备面试内容呢，于是和面试官约的时间是两天后，当时想着这两天好好刷面经总可以应付过去的，万万没想到这是我人生中第一次被“羞辱”成这样的面试，以至于面试官直接和我说“你把《Java编程思想》看完十遍以后再来面试吧”，当时一度怀疑我还能不能找到实习找到工作。就在这时，救星来了，我收到了下面这个短信。

面试之后经过漫长的等待，我终于拿到了人生中第一个实习offer(感谢华为爸爸！！！)。这个时候我想到的第一件事情是我大概可以放弃考研这条路了，再不济也可以实习的时候好好表现争取转正。正好也快到期末考试了，所以我就专心准备期末考试，争取保上研，实在不行的话就暑假的时候好好实习争取转正。

暑假实习&夏令营

时间过得很快，一眨眼就到了暑假，因为学期初给自己埋的坑，我7月份的时候还要在长沙参加“湘江人工智能学院”的实习，上半个月是在湖大进行集中的授课，老师讲的内容都有点泛泛而谈（你们就不能教一下coding嘛=_=）不过也正好省去了我参加夏令营时候的请假流程。下半个月是在一家AI独角兽企业——湘江地平线实习（但是也啥都没干。。。），不过有一说一，地平线的办公条件真的太舒服了！！幸运的是刚好碰到地平线创始人余凯博士来长沙，顺便和我们进行了相关的交流，这是我第一次和真正站在行业最前沿的大佬面对面地进行交流。了解到他作为地平线掌舵人的担当与使命，致力于将AI技术进行落地应用。

7月份另外一个重要的事情就是夏令营，花了不少力气去准备，因为是跨专业，问了身边很多学长学姐，都没有可以借鉴的例子。只能硬着头皮上，因为当时成绩没有优势，甚至于我自己都对我能否保研心存怀疑，所以报的学校不算多，而且有回应的学校也是意料之内的少。最后去了夏令营的学校只有湖大和信工所（暴风哭泣）去北京的时候见到了很久没见的高中同学，也算是意外的收获吧。下面这张图是去北航找同学的时候拍的，顺便感谢一下龙哥收留省了一晚上住宿费hhh

Java NIO

发表于 2020-01-18

NIO

NIO全称（Non-blocking IO），在Java中也叫作New IO。

IO与NIO的区别

传统IO NIO
面向流 面向缓冲区
阻塞IO 非阻塞IO
无 选择器

图1 IO的工作方式(水流式)
在连接之中传输的就是数据，像水流一样，是单向的（一个流必须是InputStream或者OutputStream的子类）

图2 NIO工作方式(火车式)
通道channel负责传输（像铁路），缓冲区Buffer负责存储（像火车），数据在通道中是可以双向流动的。

通道与缓冲区

缓冲区类型

根据数据类型不同，提供相应的缓冲区

  • ByteBuffer
  • CharBuffer
  • ShortBuffer
  • IntBuffer
  • LongBuffer
  • FloatBuffer
  • DoubleBuffer

通道

所有数据都通过 Buffer 对象来处理。您永远不会将字节直接写入通道中，相反，您是将数据写入包含一个或者多个字节的缓冲区。

同样，您不会直接从通道中读取字节，而是将数据从通道读入缓冲区，再从缓冲区获取这个字节。

缓冲区Buffer

Buffer的几个属性

  • position：当前位置
  • limit：
  • capacity：容量
  • mark：标记，表示记录当前position的位置。可以通过reset()恢复到mark的位置

Buffer缓冲区的数据操作

下面这张图说明了部分过程中Buffer属性的变化过程

直接缓冲区与非直接缓冲区

  • 非直接：通过allocate()方法分配缓冲区，将缓冲区建立在JVM的内存中
  • 直接：通过allocateDirect()方法分配直接缓冲区，将缓冲区建立在物理内存中。可以提高效率

通道Channel

传统的IO模式（重点：DMA）

NIO模式（重点：通道）
这种情况下通道是一个专门负责IO的“CPU”，不会给CPU造成负担

通道：

在Java NIO中负责缓冲区中数据的传输。Channel本身并不存储数据，因此需要配合缓冲区进行传输。

通道的主要实现类

java.nio.channels.Channel接口：
|–FileChannel(文件)
|–SocketChannel
|–ServerSocketChannel
|–DatagramChannel(UDP网络通信)
